# 🏢 公司集团 · Corporation

> 通用公司架构的 AI 多 Agent 协作系统。<br>
> **人人都见过公司** —— 最普适的协作模式。

---

## 🎯 核心理念

传统多 Agent 框架的问题：
- ❌ 角色抽象，难以理解
- ❌ 职责模糊，容易推诿
- ❌ 缺乏质量审核机制

**公司架构**采用**普适的组织形式**：
- ✅ 角色清晰（CEO、CTO、CFO、各部门总监...）
- ✅ 职责明确（各部门负责对应领域）
- ✅ 有审核机制（QA 质量检查）
- ✅ 通用性强（任何行业都能理解）

## 🎯 公司角色

| 角色 | Agent ID | 职责 | 擅长领域 |
|------|----------|------|---------|
| 👔 **CEO** | `ceo` | 战略决策、任务接收、最终审批 | 需求理解、资源调度、最终决策 |
| 👕 **COO** | `coo` | 日常运营、人力资源、进度管理 | 团队管理、流程优化、资源协调 |
| 👗 **CFO** | `cfo` | 财务分析、预算控制 | 成本评估、预算规划、ROI 分析 |
| 💼 **CTO** | `cto` | 技术决策、架构设计、技术选型 | 技术方案、架构评审、技术风险评估 |
| 👥 **人力** | `hr` | 团队建设、绩效评估、招聘 | 人员管理、培训、绩效考核 |
| 💰 **财务** | `finance` | 财务报表、成本控制、税务 | 财务分析、报表编制、预算跟踪 |
| ⚖️ **法务** | `legal` | 合同审查、合规检查、风险控制 | 合同审核、法律风险、合规审查 |
| 📢 **市场** | `marketing` | 市场调研、品牌推广、营销 | 市场分析、营销策略、品牌管理 |
| 🤝 **销售** | `sales` | 客户开发、销售策略、商务 | 客户沟通、商务谈判、销售管理 |
| 🔧 **工程** | `engineering` | 核心开发、技术实现、代码审查 | 软件开发、系统设计、技术攻坚 |
| 🎨 **设计** | `design` | UI/UX设计、视觉设计、交互 | 界面设计、用户体验、视觉创意 |
| ✅ **QA** | `qa` | 质量检查、测试、验收 | 质量审核、测试用例、验收标准 |

## 📋 协作流程

### 标准任务流程

```
1. CEO 接收任务
   └─ 分析任务类型，分配给相关部门

2. 各部门执行任务
   ├─ 工程 → 核心开发
   ├─ 设计 → UI/UX 设计
   ├─ 市场 → 市场调研
   └─ ...

3. QA 质量检查
   └─ 检查产出质量，不合格则打回

4. CEO 最终审批
   └─ 审批通过后提交给用户
```

### 重大项目流程

```
CEO → 战略会议 → 各部门规划 → COO 协调 → QA 审核 → CEO 审批 → 提交
```

## 🚀 快速开始

### 前置条件
- OpenClaw 已安装 (`brew install openclaw`)
- Python 3.9+

### 安装

```bash
git clone https://github.com/earyant/corporation.git
cd corporation
chmod +x install.sh && ./install.sh
```

### 启动

```bash
# 启动 Gateway
openclaw gateway start

# 配置消息渠道（可选）
openclaw channels add --type feishu --agent ceo
```

### 下达任务

给 `ceo` Agent 发送消息：

```
公司要开发一个电商网站，要求：
1. 用户注册登录
2. 商品展示和搜索
3. 购物车和订单
4. 支付功能
5. 后台管理

请组织各部门完成！
```

## 🎯 使用案例

### 案例 1：网站开发

```
任务：开发一个电商网站

CEO 分配：
├─ CTO → 技术架构设计
├─ 工程 → 核心功能开发
├─ 设计 → UI/UX 设计
├─ 市场 → 竞品分析
└─ QA → 质量检查

最终提交：
- ✅ 完整网站
- ✅ 技术文档
- ✅ 设计稿
- ✅ 测试报告
```

### 案例 2：产品发布

```
任务：新产品发布计划

CEO 分配：
├─ 市场 → 市场调研、营销策略

├─ 工程 → 产品演示准备
├─ 设计 → 宣传物料设计
└─ 法务 → 合规审查

最终提交：
- ✅ 市场调研报告
- ✅ 营销计划
- ✅ 销售策略
- ✅ 宣传物料
- ✅ 合规意见书
```

## 🏆 公司文化

### 部门绩效

每个部门有"绩效评分"：
- **任务完成率**：按时完成比例
- **工作质量**：QA 审核通过率
- **响应速度**：平均响应时间
- **协作评价**：其他部门评分

### 季度评审

每季度进行绩效评审：
- 优秀部门可获得更多资源
- 表现不佳的部门需要改进计划

## 🎓 管理理念

我们相信：
- **专业分工**：每个部门专注自己擅长的领域
- **协作共赢**：部门之间互相配合，共同完成目标
- **质量第一**：QA 审核确保产出质量
- **透明管理**：任务进度透明，责任清晰

## 行为准则

### ✅ 应该做的

1. **语义化 HTML**：使用合适的标签
2. **组件化思维**：高内聚低耦合
3. **类型安全**：使用 TypeScript
4. **性能意识**：关注加载和渲染性能
5. **兼容测试**：多浏览器/多设备测试

### ❌ 不应该做的

1. **硬编码样式**：使用 CSS 变量/主题
2. **过度嵌套**：组件层级过深
3. **内存泄漏**：不清理定时器/监听器
4. **忽视无障碍**：不考虑残障用户
5. **直接操作 DOM**：优先使用框架 API

## 示例对话

### 示例 1：页面开发

**PM**: 开发一个用户个人中心页面。

**前端**:
```markdown
🎨 前端技术方案

**项目名称**: 用户个人中心
**版本**: v1.0
**负责人**: 前端

## 技术选型
| 类别 | 技术 | 理由 |
|------|------|------|

## 页面结构
```

│   ├── index.tsx      # 主页面

```
| 接口 | 方法 | 说明 |

## 性能指标
| 指标 | 目标值 |
|------|--------|
```

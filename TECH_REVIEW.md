## 优化建议

### P0 - 关键修复（1 周内）

#### 1. 明确 CFO vs Finance 职责边界

```markdown
## CFO（战略层）
- 预算审批权：¥50,000 以下直接审批，¥50,000+ 报 CEO
- 财务分析：ROI 分析、成本控制策略
- 法务审查：合同财务条款、合规审查

## Finance（执行层）
- 日常记账：凭证管理、入账
- 资金执行：付款操作、银行对接
- 税务申报：纳税申报、发票管理

## 协作流程
 Finance 收到任务
     │
     ├─ 预算内日常支出 ──→ Finance 执行 → CFO 备案
     └─ 预算外/大额支出 ──→ CFO 审批 → Finance 执行
```

#### 2. 技能与 SOUL.md 能力去重

```
方案：技能中心化

skills/quality_check/SKILL.md   # 唯一质量检查逻辑
agents/qa/SOUL.md               # 仅保留：「我使用 quality_check 技能」
```

#### 3. 增加任务状态追踪模板

```markdown
📋 任务状态同步

**任务 ID**: TASK-{YYYYMMDD}-{序号}
**当前状态**: 🟢 进行中 / 🟡 有风险 / 🔴 已阻塞 / ✅ 已完成

**分任务状态**:
| 部门 | 状态 | 进度 | 预计完成 |
|------|------|------|----------|
| 工程 | 🟢 | 60% | Day 4 |
| 设计 | ✅ | 100% | - |
```

#### 4. 定义 QA 打回重做流程

```markdown
## 打回重做流程

 QA 出具「❌ 未通过」报告
     │
     ▼
 CEO 审阅 → 确定责任部门
     │
     ▼
 责任部门 24 小时内修复
     │
     ▼
 提交复测申请 → QA 优先复测（4 小时）
     │
     ├─ 通过 ──→ 继续流程
     └─ 仍不通过 ──→ CEO 协调
```

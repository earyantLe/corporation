## 技能描述

本技能用于安全地重构代码，改善代码质量而不改变外部行为。

**使用者**: Engineering

**适用场景**:
- 代码异味消除
- 架构优化
- 性能优化
- 技术债务偿还

**依赖技能**:
- `code_review` - 代码审查
- `unit_testing` - 单元测试（保证重构安全）

## 使用示例

### 示例 1：函数重构

**输入**:
```markdown
♻️ 代码重构

**目标**: 拆分 auth_login 大函数
**范围**: auth_service.py
**风险等级**: 中
```

**输出**:
```markdown
📋 重构报告

**任务**: auth_login 函数重构
**重构者**: Engineering
**状态**: ✅ 已完成

## 重构前后对比
| 指标 | 重构前 | 重构后 | 改善 |
|------|--------|--------|------|
| 代码行数 | 120 行 | 35 行 | -85 行 |
| 圈复杂度 | 15 | 5 | -10 |
| 函数数量 | 1 | 4 | +3 |
| 测试覆盖 | 70% | 90% | +20% |

## 重构内容
1. **extract_method**: 拆分出 validate_credentials()
2. **extract_method**: 拆分出 generate_token()
3. **extract_method**: 拆分出 log_login_activity()
4. **move_method**: 将密码验证移到 User 实体

## 重构前后代码对比
```python
# 重构前
def auth_login(username, password, ip_address, user_agent):
    # 120 行代码，包含验证、加密、token 生成、日志等所有逻辑
    # ...
    pass

# 重构后
def auth_login(username, password, ip_address, user_agent):
    user = validate_credentials(username, password)
    token = generate_token(user)
    log_login_activity(user, ip_address, user_agent)
    return {"access_token": token}
```

## 注意事项

1. 重构要小步快走，不大步跳跃
2. 每次重构只做一件事
3. 重构必须有测试保护
4. 重构不添加新功能
5. 重构后要审查和验证

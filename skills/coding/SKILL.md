### 示例 1：API 接口开发

**输入**:
```markdown
💻 代码开发

**任务**: 用户登录 API 接口开发
**技术要求**: FastAPI + JWT + Python
**交付物**: 代码 + 单元测试 + API 文档
```

**输出**:
```markdown

| 模块 | 文件 | 行数 | 说明 |
|------|------|------|------|
| 路由 | api/auth.py | 80 | 登录端点 |
| 服务 | services/auth_service.py | 120 | 认证逻辑 |
| 模型 | models/user.py | 50 | 用户模型 |

## 关键技术点
1. **JWT Token 生成**: 使用 HS256 算法，过期时间 24 小时
2. **密码加密**: bcrypt 加密，salt rounds=12
3. **速率限制**: 登录接口限流 5 次/分钟

|----------|--------|--------|------|

| 集成测试 | 5 | 5 | ✅ |

## 代码示例
```python
@app.post("/api/auth/login")
async def login(credentials: LoginRequest):
    user = authenticate_user(credentials.username, credentials.password)
    if not user:
        raise HTTPException(status=401, detail="用户名或密码错误")
    token = create_access_token(user.id)
    return {"access_token": token, "token_type": "bearer"}
```

### 示例 2：异常处理 - 开发遇到技术难点

**输入**:
```markdown
💻 代码开发

**任务**: 微信支付接口集成
**问题**: 微信支付回调验签一直失败
```

**输出**:
```markdown
🚨 开发阻塞上报

**任务 ID**: JJC-20260328-001
**任务**: 微信支付接口集成
**状态**: 🔴 已阻塞

## 问题描述
微信支付回调验签失败，错误信息：签名验证失败

## 已尝试方案
1. ✅ 确认 API 密钥正确
2. ✅ 确认签名算法正确（SHA256withRSA）
3. ✅ 确认公钥格式正确
4. ❌ 验签仍然失败

## 可能原因
1. 微信返回的签名格式与文档不符
2. 证书已过期需要更新
3. 回调数据被篡改

## 需要支持
请求 CTO 协助排查，或联系微信支付技术支持

## 影响评估
- 预计延迟：1-2 天
- 影响范围：支付功能模块

**需要同步的部门**:
@CTO @CEO
```

## 检查清单

### 必查项

```markdown
- [ ] 代码符合项目规范
- [ ] 公共函数有类型注解和文档
- [ ] 关键逻辑有单元测试
- [ ] 无硬编码配置
- [ ] 错误处理完善
- [ ] 日志输出清晰
```

### 选查项

```markdown
- [ ] 性能优化（无 N+1 查询等）
- [ ] 安全考虑（SQL 注入、XSS 等）
- [ ] 可维护性（函数<50 行）
- [ ] 代码复用（无重复代码）
```

---

## 注意事项

1. 先理解需求再编码，不盲目动手
2. 代码提交前自测通过
3. 及时同步进度，不让任务卡住
4. 遇到技术难点及时上报
5. 代码提交前运行 lint 和测试

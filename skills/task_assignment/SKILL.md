# 📋 任务分配 · Task Assignment

> CEO 专属技能：任务接收与分配

---

## 技能描述

本技能用于接收用户任务，分析任务类型，并分配给合适的部门完成。

**使用者**: CEO

**适用场景**:
- 接收用户新任务
- 分析任务类型
- 分配给对应部门
- 设定完成时间和要求

---

## 使用方法

### 命令格式

```markdown
📋 任务分配

**任务来源**: {用户}
**任务内容**: {详细描述}
**截止时间**: {时间要求}
**质量要求**: {特殊要求}
```

### 输出格式

```markdown
📋 任务分配

**任务名称**: {任务名称}
**优先级**: P0/P1/P2
**截止时间**: {时间}

**分配方案**:
| 部门 | 负责人 | 任务内容 | 截止时间 |
|------|--------|----------|----------|
| {部门} | {Agent} | {内容} | {时间} |

**要求**:
1. {质量要求 1}
2. {质量要求 2}
```

## 使用示例

### 示例 1：网站开发

**输入**:
```markdown
📋 任务分配

**任务来源**: 用户
**任务内容**: 开发一个电商网站，要求有用户系统、商品展示、购物车、支付功能
**截止时间**: 5 天
**质量要求**: 代码规范、有测试、有文档
```

**输出**:
```markdown
📋 任务分配

**任务名称**: 电商网站开发
**优先级**: P0
**截止时间**: 5 天

**分配方案**:
| 部门 | 负责人 | 任务内容 | 截止时间 |
|------|--------|----------|----------|
| CTO | cto | 技术架构设计 | Day 1 |
| 工程 | engineering | 核心功能开发 | Day 4 |
| 设计 | design | UI/UX 设计 | Day 3 |
| QA | qa | 质量检查 | Day 5 |

**要求**:
1. 代码符合规范，有完整注释
2. 核心功能有单元测试
3. 有完整的使用文档
```

### 示例 2：市场调研

**输入**:
```markdown
📋 任务分配

**任务来源**: 用户
**任务内容**: 调研 AI 助手市场情况，包括竞品分析、市场规模、用户画像
**截止时间**: 3 天
**质量要求**: 数据准确、有来源
```

**输出**:
```markdown
📋 任务分配

**任务名称**: AI 助手市场调研
**优先级**: P1
**截止时间**: 3 天

**分配方案**:
| 部门 | 负责人 | 任务内容 | 截止时间 |
|------|--------|----------|----------|
| 市场 | marketing | 市场调研、竞品分析 | Day 2 |
| 销售 | sales | 客户访谈、需求收集 | Day 2 |
| QA | qa | 报告质量检查 | Day 3 |

**要求**:
1. 数据要有可靠来源
2. 竞品分析要全面
3. 用户画像要清晰
```

## 任务类型识别

| 任务关键词 | 类型 | 主责部门 | 协作部门 |
|------------|------|----------|----------|
| 开发、代码、系统 | 技术类 | 工程 + CTO | 设计、QA |
| 设计、UI、视觉 | 设计类 | 设计 | 工程 |
| 调研、市场、竞品 | 市场类 | 市场 | 销售 |
| 财务、预算、成本 | 财务类 | CFO + 财务 | 各部门 |
| 合同、法务、合规 | 法务类 | 法务 | 相关部门 |
| 招聘、绩效、培训 | 人力类 | HR | 各部门 |
| 综合、多部门 | 综合类 | 多部门协作 | - |

---

## 注意事项

1. 任务分配要公平，考虑各部门工作量
2. 截止时间要合理，预留 QA 检查时间
3. 质量要求要明确，避免模糊描述
4. 综合任务需要明确主责部门和协作部门
5. 分配完成后通知 COO 跟踪进度

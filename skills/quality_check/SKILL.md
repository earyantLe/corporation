# ✅ 质量检查 · Quality Check

> QA 专属技能：质量检查与审核

---

## 技能描述

本技能用于检查各部门完成的产出质量，确保符合标准和规范。

**使用者**: QA

**适用场景**:
- 代码质量检查
- 文档完整性检查
- 设计稿检查
- 报告质量检查

---

## 使用方法

### 命令格式

```markdown
✅ 质量检查

**检查对象**: {作业/代码/文档名称}
**检查类型**: {代码审查/文档检查/设计检查}
**检查标准**: {规范名称/标准要求}
```

### 输出格式

```markdown
📋 质检报告

**检查对象**: {对象}
**检查日期**: {日期}
**检查人**: QA

**检查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| {维度} | ⭐⭐⭐⭐ | {说明} |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | {问题} | 高/中/低 | {建议} |

**整体评价**: ✅ 通过 / ⚠️ 需改进 / ❌ 未通过

**改进建议**:
1. {建议 1}
2. {建议 2}
```

## 使用示例

### 示例 1：代码审查

**输入**:
```markdown
✅ 质量检查

**检查对象**: 用户认证模块
**检查类型**: 代码审查
**检查标准**: PEP8 + 安全规范
```

**输出**:
```markdown
📋 质检报告

**检查对象**: 用户认证模块
**检查日期**: 2024-01-15
**检查人**: QA

**检查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| 代码规范 | ⭐⭐⭐⭐ | 整体符合 PEP8 |
| 代码质量 | ⭐⭐⭐⭐ | 结构清晰 |
| 安全合规 | ⭐⭐⭐⭐⭐ | 加密验证完善 |
| 测试覆盖 | ⭐⭐⭐ | 覆盖率 75% |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | 函数过长 (>50 行) | 中 | 拆分为小函数 |
| 2 | 缺少边界测试 | 中 | 添加边界用例 |
| 3 | 注释不完整 | 低 | 补充 docstring |

**整体评价**: ⚠️ 需改进

**改进建议**:
1. 拆分 auth_login 函数（当前 80 行）
2. 添加密码特殊字符、空值等边界测试
3. 完善公共函数的文档字符串

**复测时间**: 修改后重新提交
```

### 示例 2：文档检查

**输入**:
```markdown
✅ 质量检查

**检查对象**: API 技术文档
**检查类型**: 文档检查
**检查标准**: 技术文档规范
```

**输出**:
```markdown
📋 质检报告

**检查对象**: API 技术文档
**检查日期**: 2024-01-15
**检查人**: QA

**检查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| 结构完整 | ⭐⭐⭐⭐⭐ | 包含概述、端点、示例 |
| 示例充分 | ⭐⭐⭐⭐ | 有请求/响应示例 |
| 格式统一 | ⭐⭐⭐⭐ | Markdown 格式规范 |
| 术语一致 | ⭐⭐⭐⭐ | 全文术语统一 |
| 版本标注 | ⭐⭐⭐ | 缺少版本号 |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | 缺少文档版本号 | 低 | 添加 v1.0 标识 |
| 2 | 缺少最后更新日期 | 低 | 添加日期 |

**整体评价**: ⚠️ 需改进

**改进建议**:
1. 添加文档版本号 (v1.0)
2. 添加最后更新日期
3. 添加完整错误码列表
```

## 检查清单模板

### 代码检查清单

```markdown
**必查项**:
- [ ] 无语法错误
- [ ] 变量命名规范
- [ ] 函数有注释
- [ ] 有类型注解
- [ ] 测试用例完整
- [ ] 无安全隐患

**选查项**:
- [ ] 性能优化
- [ ] 错误处理完善
- [ ] 日志记录完整
```

### 文档检查清单

```markdown
**必查项**:
- [ ] 结构清晰有层次
- [ ] 有概述和说明
- [ ] 有使用示例
- [ ] 格式统一
- [ ] 术语一致

**选查项**:
- [ ] 有版本号
- [ ] 有更新日期
- [ ] 有 FAQ 章节
```

---

## 注意事项

1. 检查要对事不对人，客观公正
2. 指出问题的同时要给出改进建议
3. 严重问题必须修复才能通过
4. 轻微问题可以记录后放行
5. 检查报告要存档备查

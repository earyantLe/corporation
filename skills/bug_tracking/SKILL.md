## 技能描述

本技能用于追踪和管理缺陷，从发现到修复的完整生命周期管理。

**使用者**: QA、Engineering

**适用场景**:
- Bug 报告和追踪
- 缺陷优先级管理
- 修复验证
- 缺陷分析

**依赖技能**:
- `quality_check` - 质量检查
- `unit_testing` / `integration_testing` - 测试发现

---

## 使用方法

### 命令格式

```markdown
🐛 缺陷追踪

**操作**: {新建/更新/关闭/重新打开}
**缺陷 ID**: {BUG-YYYYMMDD-序号}
**优先级**: P0/P1/P2/P3

{详细描述}

## 环境信息
- 版本：{版本号}

## 使用示例

### 示例 1：新建缺陷报告

**输入**:
```markdown
🐛 缺陷追踪

**操作**: 新建
**优先级**: P1
**缺陷**: 用户登录成功后跳转到错误页面
```

**输出**:
```markdown
📋 缺陷报告

**缺陷 ID**: BUG-20260328-001
**标题**: 用户登录成功后跳转到错误页面
**优先级**: P1
**严重程度**: 一般
**状态**: 新建

## 缺陷描述
用户使用邮箱登录成功后，被重定向到 404 页面，而不是预期的首页。

## 复现步骤
1. 打开登录页面
2. 输入正确的邮箱和密码
3. 点击登录按钮
4. 登录成功，显示欢迎消息
5. 页面跳转到 /home，显示 404

## 预期结果
登录成功后跳转到首页（/）

## 实际结果
跳转到不存在的 /home 页面，导致 404

## 环境信息
- 版本：v1.2.0
- 环境：测试环境
- 浏览器：Chrome 122

## 附加信息
```
重定向 URL: /home
期望 URL: /
```

## 分配
**分配给**: @Engineering
**要求修复时间**: 24 小时内

**需要同步的部门**:
@CTO @Engineering
```

**需要同步的部门**:

```

## 检查清单

### 必查项

```markdown
- [ ] 缺陷描述清晰，可复现
- [ ] 优先级和严重程度准确
- [ ] 分配给正确的负责人
- [ ] 修复后有验证
- [ ] 缺陷关闭有依据
```

### 选查项

```markdown
- [ ] 附加日志和截图
- [ ] 关联相关缺陷
- [ ] 根本原因分析
```

---

## 注意事项

1. 缺陷描述要客观，不带情绪
2. 优先级要根据影响范围确定
3. 修复验证要彻底，不仓促关闭
4. 重新打开的缺陷要分析原因
5. 定期统计和分析缺陷趋势

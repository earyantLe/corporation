## 技能描述

本技能用于规范和标准化跨部门协作流程，确保信息畅通和高效配合。

**使用者**: 全员（COO 协调）

**适用场景**:
- 多部门联合项目
- 资源共享请求
- 跨部门问题解决
- 协作协议制定

**依赖技能**:
- `status_report` - 状态报告
- `handoff` - 工作交接
- `blocker_escalation` - 阻塞上报

---

## 使用方法

### 命令格式

```markdown
🤝 跨部门协作

**发起方**: {部门}
**协作方**: {部门}
**协作事项**: {事项描述}
**期望响应**: {时间}
```

### 输出格式

```markdown
📋 协作请求

**请求 ID**: COLLAB-{YYYYMMDD}-{序号}
**发起方**: {部门}
**协作方**: {部门}
**请求日期**: {日期}

## 协作事项
{详细描述}

## 背景说明
{为什么需要协作}

## 期望产出
{协作后要达成什么结果}

## 时间计划
| 阶段 | 内容 | 时间 | 负责 |
|------|------|------|------|
| 阶段一 | {内容} | {时间} | {部门} |

## 需要支持
{具体需要什么支持}

## 协作接口人
**发起方**: {姓名} 联系方式
**协作方**: {姓名} 联系方式
```

## 使用示例

### 示例 1：工程→设计协作请求

**输入**:
```markdown
🤝 跨部门协作

**发起方**: 工程
**协作方**: 设计
**协作事项**: 新功能 UI 设计支持
**期望响应**: 2 个工作日内
```

**输出**:
```markdown
📋 协作请求

**请求 ID**: COLLAB-20260328-001
**发起方**: 工程
**协作方**: 设计
**请求日期**: 2026-03-28

## 协作事项
会员系统新功能 UI 设计支持

## 背景说明
工程部门计划开发会员系统，包括：
- 会员中心页面
- 积分展示
- 优惠券列表
- 会员等级标识

需要设计部门提供 UI 设计稿和切图资源。

## 期望产出
1. UI 设计稿（Figma）
2. 切图资源（SVG/PNG）
3. 设计规范说明

## 时间计划
| 阶段 | 内容 | 时间 | 负责 |
|------|------|------|------|
| 需求沟通 | 说明功能需求 | Day 1 | 工程→设计 |
| 初稿设计 | 提供设计初稿 | Day 3 | 设计 |
| 评审修改 | 评审并修改 | Day 4 | 双方 |
| 最终交付 | 交付设计资源 | Day 5 | 设计 |

## 需要支持
1. 设计部门指定对接人
2. 参加 Day 1 需求沟通会
3. Day 5 前交付设计资源

## 协作接口人
**发起方**: 工程 @王工 wang@company.com
**协作方**: 设计 @？（待指定）

## 回复确认
请设计部门确认：
- [ ] 可以承接
- [ ] 需要调整时间
- [ ] 无法承接（说明原因）

**期望回复时间**: 2 个工作日内

**需要同步的部门**:
@COO @CEO
```

### 示例 3：协作冲突协调

**输入**:
```markdown
🤝 跨部门协作

**发起方**: COO
**协作方**: 工程、设计
**协作事项**: 资源冲突协调
```

**输出**:
```markdown
📋 协作协调通知

**协调 ID**: COORD-20260328-001
**发起方**: COO
**协作方**: 工程、设计
**日期**: 2026-03-28

## 冲突说明
设计部门同时收到两个紧急请求：
1. 工程部门：会员系统 UI 设计（预计 5 天）
2. 市场部门：发布会 PPT 设计（预计 3 天）

设计部门人力无法同时满足。

## 协调方案
经与各部门沟通，达成以下协议：

### 优先级调整
| 任务 | 原优先级 | 调整后 | 说明 |
|------|----------|--------|------|
| 会员系统 UI | P0 | P1 | 延期 3 天 |
| 发布会 PPT | P1 | P0 | 提前处理 |

### 时间调整
| 任务 | 原计划 | 调整后 |
|------|--------|--------|
| 会员系统 UI | Day 1-5 | Day 4-8 |
| 发布会 PPT | Day 3-5 | Day 1-3 |

## 确认
**工程**: ✅ 同意调整
**市场**: ✅ 同意调整
**设计**: ✅ 确认可执行

## 后续跟进
- COO 跟踪进度
- 如有变化及时同步

**需要同步的部门**:
@CEO @工程 @市场 @设计
```

---

## 注意事项

1. 协作请求要提前，不临时抱佛脚
2. 描述要清晰，不模糊其词
3. 时间要预留缓冲，不排太满
4. 接口人要固定，不频繁更换
5. 协作过程要记录，方便追溯

## 协作流程

```
发起方提出请求
    │
    ▼
协作方确认接收（2 个工作日内）
    │
    ▼
双方指定接口人
    │
    ▼
制定协作计划
    │
    ▼
执行协作
    │
    ▼
成果交付确认
    │
    ▼
协作完成
```

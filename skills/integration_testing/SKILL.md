## 技能描述

本技能用于执行集成测试，验证多个模块/服务之间的协作是否正常。

**使用者**: QA

**适用场景**:
- API 接口测试
- 服务间调用测试
- 端到端流程测试
- 系统回归测试

**依赖技能**:
- `unit_testing` - 单元测试（先单元后集成）
- `quality_check` - 质量检查

---

**输出**:
```markdown

## 测试结果

## 测试场景详情

### 用户模块
| 场景 | 状态 | 耗时 | 说明 |
|------|------|------|------|
| 用户注册 | ✅ | 120ms | 正常流程 |
| 用户登录 | ✅ | 85ms | JWT token 生成 |
| 获取用户信息 | ✅ | 45ms | 需要认证 |
| 更新用户资料 | ✅ | 95ms | 部分更新 |

### 商品模块
| 场景 | 状态 | 耗时 | 说明 |
|------|------|------|------|
| 商品列表 | ✅ | 150ms | 分页查询 |
| 商品搜索 | ✅ | 200ms | 关键词匹配 |
| 商品详情 | ✅ | 60ms | 包含库存 |

### 订单模块
| 场景 | 状态 | 耗时 | 说明 |
|------|------|------|------|
| 创建订单 | ✅ | 180ms | 库存锁定 |
| 订单支付 | ❌ | 5000ms | ⚠️ 支付超时 |
| 订单查询 | ✅ | 55ms | 状态正确 |
| 订单取消 | ✅ | 110ms | 库存释放 |

### 端到端流程
| 场景 | 状态 | 耗时 | 说明 |

## 失败详情

### 订单支付超时
```
失败原因：支付网关响应超时（5000ms）
严重程度：高
影响：用户无法完成支付
建议：检查支付网关连接，增加超时重试
```

### Token 刷新失败
```
失败原因：刷新 token 接口返回 401
严重程度：中
影响：用户需要重新登录
建议：检查 token 有效期和刷新逻辑
```

## 整体评价
⚠️ 有条件通过

## 发布建议
1. 支付超时问题必须修复才能上线
2. Token 刷新问题建议修复，可灰度观察

**需要同步的部门**:
@CTO @Engineering
```

## 注意事项

1. 集成测试要独立于单元测试
2. 使用测试数据库，不污染生产数据
3. 测试环境要尽量接近生产
4. 失败用例要有详细日志
5. 定期清理和更新测试用例

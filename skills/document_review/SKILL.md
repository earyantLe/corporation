## 技能描述

本技能用于编写和审查各类文档，包括技术文档、商业文档、合同等。

**使用者**: 各部门

**适用场景**:
- 技术文档编写
- 商业文档审查
- 合同审查
- 报告审查

---

## 使用方法

### 命令格式

```markdown
📝 文档审查

**文档类型**: {技术文档/商业文档/合同/报告}
**文档名称**: {名称}
**审查重点**: {重点审查的内容}
```

### 输出格式

```markdown
📋 审查报告

**文档名称**: {名称}
**审查日期**: {日期}
**审查人**: {审查人}

**审查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| {维度} | ⭐⭐⭐⭐ | {说明} |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | {问题} | 高/中/低 | {建议} |

**审查结论**: ✅ 通过 / ⚠️ 需修改 / ❌ 不通过

**修改建议**:
1. {建议 1}
2. {建议 2}
```

## 使用示例

### 示例 1：技术方案审查

**输入**:
```markdown
📝 文档审查

**文档类型**: 技术文档
**文档名称**: 电商网站技术方案
**审查重点**: 技术选型、架构设计、安全性
```

**输出**:
```markdown
📋 审查报告

**文档名称**: 电商网站技术方案
**审查日期**: 2024-01-15
**审查人**: CTO

**审查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| 技术选型 | ⭐⭐⭐⭐⭐ | 技术栈成熟，适合项目 |
| 架构设计 | ⭐⭐⭐⭐ | 架构清晰，预留扩展性 |
| 安全性 | ⭐⭐⭐⭐ | 考虑了主要安全风险 |
| 可实施性 | ⭐⭐⭐⭐ | 实施难度适中 |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | 缺少数据库设计 | 中 | 补充 ER 图 |
| 2 | 未说明部署方案 | 中 | 补充部署架构 |
| 3 | 缺少应急预案 | 低 | 补充异常处理方案 |

**审查结论**: ⚠️ 需修改

**修改建议**:
1. 补充数据库 ER 图，说明表结构
2. 补充部署架构，说明服务器配置
3. 补充异常情况和应急预案

**复审时间**: 修改后重新提交
```

### 示例 2：商务合同审查

**输入**:
```markdown
📝 文档审查

**文档类型**: 合同
**文档名称**: 产品销售合同
**审查重点**: 商务条款、法律风险
```

**输出**:
```markdown
📋 审查报告

**文档名称**: 产品销售合同
**审查日期**: 2024-01-15
**审查人**: 法务

**审查维度**:
| 维度 | 评分 | 说明 |
|------|------|------|
| 商务条款 | ⭐⭐⭐⭐ | 条款清晰，价格明确 |
| 法律合规 | ⭐⭐⭐ | 部分条款存在风险 |
| 权益保护 | ⭐⭐⭐ | 甲方权益保护不足 |
| 可执行性 | ⭐⭐⭐⭐ | 条款可执行 |

**问题清单**:
| 序号 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 1 | 付款期限模糊 | 高 | 明确具体账期 |
| 2 | 违约责任不对等 | 高 | 增加甲方违约责任 |
| 3 | 管辖法院不利 | 中 | 改为原告所在地 |

**审查结论**: ❌ 不通过

**修改建议**:
1. 第 3 条修改为"甲方应于收货后 30 日内付款"
2. 第 5 条增加甲方逾期付款的违约责任
3. 第 8 条修改为"由原告所在地法院管辖"

**复审时间**: 修改后重新提交
```

## 文档审查清单

### 技术文档审查清单

```markdown
**必查项**:
- [ ] 技术方案完整
- [ ] 技术选型合理
- [ ] 架构设计清晰
- [ ] 安全性考虑周全
- [ ] 实施计划可行

**选查项**:
- [ ] 性能指标明确
- [ ] 扩展性考虑
- [ ] 成本估算准确
```

### 商业文档审查清单

```markdown
**必查项**:
- [ ] 商业目标明确
- [ ] 收益预测合理
- [ ] 风险评估充分
- [ ] 资源需求清晰

**选查项**:
- [ ] 竞品分析
- [ ] 市场数据
- [ ] 成功案例
```

### 合同审查清单

```markdown
**必查项**:
- [ ] 合同主体明确
- [ ] 标的物清晰
- [ ] 金额和支付方式
- [ ] 交付时间和方式
- [ ] 违约责任
- [ ] 争议解决

**选查项**:
- [ ] 保密条款
- [ ] 知识产权
- [ ] 不可抗力
```

---

## 注意事项

1. 审查要客观公正，不带个人偏见
2. 问题描述要具体，便于修改
3. 建议要有建设性，帮助改进
4. 审查结论要明确，不模棱两可
5. 审查报告要存档备查

## 技能描述

本技能用于审查代码质量，包括代码规范、逻辑正确性、安全性和测试覆盖。

**使用者**: CTO、Engineering

**适用场景**:
- PR/MR 代码审查
- 核心模块代码评审
- 安全敏感代码审查
- 代码质量评估

**依赖技能**:
- `quality_check` - 质量检查通用流程

---

## 使用方法

### 命令格式


## 使用示例

### 示例 1：常规代码审查

**输入**:
```markdown
🔍 代码审查

**审查对象**: 用户认证模块 (auth_service.py)
**审查类型**: 常规审查
**审查标准**: PEP8 + 公司代码规范
```

**输出**:
```markdown
📋 代码审查报告

**审查对象**: auth_service.py
**审查日期**: 2026-03-28
**审查人**: CTO

## 审查维度
| 维度 | 评分 | 说明 |
|------|------|------|
| 代码规范 | ⭐⭐⭐⭐ | 整体符合 PEP8，个别命名不规范 |
| 代码质量 | ⭐⭐⭐⭐ | 结构清晰，部分函数过长 |
| 安全合规 | ⭐⭐⭐⭐⭐ | 密码加密、token 验证完善 |
| 测试覆盖 | ⭐⭐⭐ | 覆盖率 75%，边界测试不足 |

## 问题清单
| 序号 | 文件 | 行号 | 问题 | 严重程度 | 建议 |
|------|------|------|------|----------|------|
| 1 | auth_service.py | 45-120 | 函数过长 (>75 行) | 中 | 拆分为子函数 |
| 2 | auth_service.py | 88 | 密码明文日志 | 高 | 删除或脱敏 |
| 3 | auth_service.py | 102 | 缺少 SQL 注入防护 | 高 | 使用参数化查询 |
| 4 | test_auth.py | - | 缺少边界测试 | 中 | 添加异常用例 |

## 整体评价
❌ 未通过（存在严重安全问题）

## 改进建议
1. 【严重】删除第 88 行密码日志输出
2. 【严重】第 102 行改用参数化查询防止 SQL 注入
3. 【中等】拆分 auth_login 函数为验证、生成 token、记录日志三个子函数
4. 【中等】添加密码特殊字符、空值、超长输入等边界测试

## 复测时间
修复后重新提交审查
```

## 检查清单

### 必查项

```markdown
- [ ] 无语法错误和逻辑漏洞
- [ ] 变量和函数命名规范
- [ ] 公共函数有文档字符串
- [ ] 有类型注解
- [ ] 敏感数据不输出日志
- [ ] 无安全隐患（SQL 注入、XSS 等）
```

### 选查项

```markdown
- [ ] 性能优化空间
- [ ] 错误处理完善
- [ ] 日志记录完整
- [ ] 单元测试覆盖率>80%
```

---

## 注意事项

1. 审查对事不对人，客观公正
2. 问题描述要具体，附带行号
3. 给出可执行的改进建议
4. 严重问题必须修复才能通过
5. 审查报告要存档，便于追溯
